Description

Ce script permet de remplir des rectangles avec des dégradés de couleur.

LinearGradient(float x, float y, float w, float h, array col1, array col2 [, array coords])

x : abscisse du coin supérieur gauche du rectangle.
y : ordonnée du coin supérieur gauche du rectangle.
w : largeur du rectangle.
h : hauteur du rectangle.
col1 : première couleur (composantes RVB).
col2 : seconde couleur (composantes RVB).
coords : tableau de la forme (x1, y1, x2, y2) qui définit la direction du dégradé (voir linear_gradient_coords.jpg). La valeur par défaut est de la gauche vers la droite (x1=0, y1=0, x2=1, y2=0).

RadialGradient(float x, float y, float w, float h, array col1, array col2 [, array coords])

x : abscisse du coin supérieur gauche du rectangle.
y : ordonnée du coin supérieur gauche du rectangle.
w : largeur du rectangle.
h : hauteur du rectangle.
col1 : première couleur (composantes RVB).
col2 : seconde couleur (composantes RVB).
coords : tableau de la forme (fx, fy, cx, cy, r) où (fx, fy) est le point de départ du dégradé de couleur 1, (cx, cy) est le centre du cercle de couleur 2, et r est le rayon du cercle (voir radial_gradient_coords.jpg).

CoonsPatchMesh(float x, float y, float w, float h, array col1, array col2, array col3, array col4 [, array coords [, float coords_min [, float coords_max]]])

x : abscisse du coin supérieur gauche du rectangle.
y : ordonnée du coin supérieur gauche du rectangle.
w : largeur du rectangle.
h : hauteur du rectangle.
col1 : première couleur (coin inférieur gauche) (composantes RVB).
col2 : deuxième couleur (coin inférieur droit) (composantes RVB).
col3 : troisième couleur (coin supérieur droit) (composantes RVB).
col4 : quatrième couleur (coin supérieur gauche) (composantes RVB).
coords :

- pour un seul patch

array(float x1, float y1, .... float x12, float y12) : 12 paires de coordonnées (de 0 à 1) qui spécifient les points de contrôle de la courbe de Bézier

- pour plusieurs patchs

array[nombre de patchs] : tableaux contenant les clés suivantes pour chaque patch :
f : position du patch (0 = premier patch, 1, 2, 3 = droite, haut ou gauche par rapport au précédent patch)
points : 12 paires de coordonnées pour les points de contrôle du premier patch, puis 8 paires pour les suivants
colors : 4 couleurs pour le premier patch, 2 pour les suivants

coords_min : valeur minimale utilisée pour les coordonnées (0 par défaut)
coords_max : valeur maximale utilisée pour les coordonnées (1 par défaut)
